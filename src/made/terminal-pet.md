<h1 class="log-post-title">terminal-pet</h1>
<p class="meta">feb 2026</p>

<div class="prose">

Inspired by VS Code pets — I just wanted something alive in my terminal.

Started simple, then I kept iterating. What if you could feed it? What if there were mini-games — hangman, tic-tac-toe? What if it had moods?

Then the idea that tied it all together: what if git commits and passing tests were what kept it alive? Write code, your pet eats. Run tests, your pet is happy. Stop committing, it starves.

So now whenever I commit, my pet gets a little happier. Simple.

*Update (Apr 2026):* Claude Code released `/buddy` — a built-in terminal pet. Built mine two months before they shipped theirs. Happy that I had the same idea as some of the best developers sitting in SF — two months before they shipped it.
